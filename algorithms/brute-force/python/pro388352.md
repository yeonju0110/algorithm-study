## 💡 풀이 과정 (생각의 흐름)

### 1단계: 문제 분석 및 알고리즘 선택
- n이 최대 30이니까 combination(30, 5) = 142,506
- m은 최대 10이니까 142,506 × 10 = 1,425,060 ≈ 10^6
- 그냥 조합으로 완탐돌려도 ㄱㅊ겠다.

### 코드

```python
from itertools import combinations

def solution(n, q, ans):
    answer = 0
    m = len(q)
    
    # 1. 조합을 구해보아요.
    # 1부터 n까지 숫자 중에서 5개를 뽑는 모든 조합 생성
    # 예: n=6이면 -> (1,2,3,4,5), (1,2,3,4,6), (1,2,3,5,6), ...
    candidates = combinations(range(1, n + 1), 5)
    
    # 각 조합(candidate)이 주어진 조건(q, ans)에 맞는지 검사
    for candidate in candidates:
        is_possible = True  # 조건을 만족하는지 여부
        
        for i in range(m):
            query = q[i]      # i번째 질의 (예: [1,3,5,7,9])
            intersection = set(candidate) & set(query)  # 조합과 질의의 교집합
            if len(intersection) != ans[i]:  # 교집합 크기가 정답(ans[i])과 다르면 불가능
                is_possible = False
                break  # 더 볼 필요 없으므로 탈출
                
        if is_possible:  # 모든 조건을 통과하면 가능한 조합으로 카운트
            answer += 1
            
    return answer  # 가능한 조합의 개수 반환

```

### 📈 복잡도 분석

#### 시간 복잡도
**O(n^5 × m)**

- n개 중 5개를 선택하는 조합: C(n, 5) = O(n^5)
- 각 조합마다 m번의 시도 검증
- 교집합 계산: O(1) (크기 5로 고정)

#### 실제 수치 (최악의 경우)
- n=30, m=10일 때
- 조합 개수: C(30, 5) = 142,506개
- 총 검증 횟수: 142,506 × 10 = **1,425,060회**
- Python에서 1초 내 처리 가능 ✅
